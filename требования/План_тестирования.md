## 1. ОБЩИЕ СВЕДЕНИЯ

### 1.1 Цель тестирования
Проверка соответствия реализованной системы требованиям ТЗ: функциональная полнота, корректность бизнес-логики, безопасность, надёжность, производительность и удобство использования.

### 1.2 Объект тестирования
Веб-приложение «Электронный журнал» (клиентская часть на React 18 + серверная часть на Node.js/Express + СУБД PostgreSQL 18).

### 1.3 Уровни тестирования
| Уровень | Тип тестов | Инструмент / Способ |
|---------|-----------|---------------------|
| Модульное (Unit) | Тесты отдельных функций, валидаторов, утилит | Jest (server), Vitest (client) |
| Интеграционное | Тесты API endpoint'ов, взаимодействие с БД | Jest + Supertest |
| Системное (E2E) | Полные пользовательские сценарии в браузере | Ручное тестирование |
| Нагрузочное | Проверка работы под нагрузкой | Ручное / Artillery |
| Приёмочное | Проверка соответствия ТЗ | Ручное + чек-лист |

### 1.4 Критерии начала тестирования
- Сервер и клиент собираются без ошибок.
- База данных развёрнута и заполнена тестовыми данными (`seeds.sql`).
- Все основные маршруты доступны.

### 1.5 Критерии окончания тестирования
- Все блокирующие (Critical) дефекты устранены.
- Покрытие критических пользовательских сценариев — 100%.
- Все пункты настоящего плана имеют статус «Пройден».
